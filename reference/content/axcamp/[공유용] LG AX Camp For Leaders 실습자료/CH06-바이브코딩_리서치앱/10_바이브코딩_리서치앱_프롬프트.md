# CH06 바이브코딩 실습: 경쟁사 리서치 대시보드

> **핵심 메시지**: "IT 부서 대기 3개월 → AI에 지시 30분"
>
> **실습 도구**: Google AI Studio (Build 모드)
> **소요 시간**: 약 20~30분

---

## 강사 노트

```
[진행 가이드]
- 초급반: 메인 프롬프트(Google Search 그라운딩)만 사용
- 상급반: 메인 프롬프트 완성 후 → 상급 프롬프트로 확장
- 프롬프트는 그대로 복사-붙여넣기 가능하도록 구성
- 수강생이 직접 회사명/산업을 바꿔 넣도록 안내
- Build 모드 진입 방법: AI Studio 좌측 메뉴 → "Build" 클릭
```

## 1. 메인 프롬프트 (초급용 - Google Search 그라운딩)

### 사전 설정

| 항목 | 설정값 |
|------|--------|
| 모델 | Gemini 2.0 Flash |
| 도구 | Google Search (그라운딩) ON |
| 모드 | Build |

### 프롬프트 (복사-붙여넣기용)

```
경쟁사 뉴스 및 기술 동향을 자동으로 수집하고 정리해주는 리서치 대시보드 웹앱을 만들어줘.

## 핵심 기능

### 1. 검색 기능
- 상단에 검색창: 회사명 또는 키워드 입력
- 검색 버튼 클릭 시 Google Search 그라운딩을 통해 최신 뉴스/기술 동향 수집
- 예시 검색어: "삼성전자 AI 전략", "현대자동차 전기차", "SK하이닉스 HBM"

### 2. 자동 요약
- 수집된 정보를 3줄 핵심 요약으로 정리
- 각 뉴스/기사별 한줄 요약 카드 표시
- 전체 동향 종합 분석 섹션 (5줄 이내)

### 3. 필터링
- 기간 필터: 최근 1주 / 1개월 / 3개월
- 카테고리 필터: 기술/제품 / 인사/조직 / 실적/재무 / 전략/M&A
- 중요도 표시: 높음/보통/낮음 (AI가 자동 판단)

### 4. 표 정리
- 수집 결과를 테이블로 정리
- 컬럼: 날짜 | 출처 | 제목 | 카테고리 | 핵심내용 | 중요도
- 테이블 정렬 기능 (날짜순, 중요도순)
- CSV 다운로드 버튼

## UI/UX 요구사항
- 깔끔한 비즈니스 대시보드 스타일
- 다크/라이트 모드 토글
- 반응형 디자인 (모바일 대응)
- 상단: 검색 영역
- 중단: 종합 요약 카드 + 카테고리별 통계 차트
- 하단: 상세 뉴스 테이블

## 기술 스택
- HTML, CSS, JavaScript (단일 페이지)
- 차트는 Chart.js 사용
- 테이블은 정렬/필터 가능하게 구현
```

## 2. 상급 프롬프트 (Firecrawl API 연동)

### 사전 준비

```
[강사 노트]
- Firecrawl API 키는 사전에 준비 (https://firecrawl.dev)
- 무료 플랜: 월 500 크레딧 (실습에 충분)
- 상급반 수강생에게 미리 API 키 발급 안내 필요
- API 키가 없는 수강생은 초급 버전으로 진행
```

| 항목 | 설명 |
|------|------|
| Firecrawl 가입 | https://firecrawl.dev → Sign Up |
| API 키 발급 | Dashboard → API Keys → Create |
| 무료 한도 | 월 500 크레딧 (페이지당 1크레딧) |

### 프롬프트 (복사-붙여넣기용)

```
아래 초급 버전 리서치 대시보드에 Firecrawl API를 연동해서
특정 웹사이트를 직접 크롤링하는 고급 기능을 추가해줘.

## 추가 기능: Firecrawl 직접 크롤링

### 1. URL 직접 크롤링
- "URL 크롤링" 탭 추가
- 사용자가 특정 URL을 입력하면 Firecrawl API로 해당 페이지 내용 추출
- 추출된 내용을 AI가 자동 요약
- 지원 URL 예시:
  - 경쟁사 IR 페이지
  - 기술 블로그
  - 보도자료 페이지
  - 산업 리포트 페이지

### 2. 사이트맵 크롤링
- 특정 도메인의 최근 게시글 목록 자동 수집
- 예: "https://newsroom.samsung.com" 입력 → 최신 보도자료 10건 자동 수집
- 수집된 내용 자동 요약 및 테이블 정리

### 3. 정기 모니터링 설정 (UI만 구현)
- 모니터링할 URL 목록 관리
- 크롤링 주기 설정 UI (실제 스케줄링은 백엔드 필요하므로 UI만)
- "지금 실행" 버튼으로 수동 크롤링

### Firecrawl API 설정
- API 키 입력 필드 (설정 패널)
- API 키는 로컬스토리지에 저장
- 연결 테스트 버튼

### API 호출 예시 (참고용)
```javascript
// Firecrawl API 호출
const response = await fetch('https://api.firecrawl.dev/v1/scrape', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_API_KEY'
  },
  body: JSON.stringify({
    url: '크롤링할_URL',
    formats: ['markdown']
  })
});
```

### UI 변경사항
- 상단 탭 추가: [Google 검색] [URL 크롤링] [모니터링]
- 설정 아이콘(⚙️) → API 키 관리 패널
- URL 크롤링 결과도 동일한 테이블 형식으로 표시
```

## 3. 초급/상급 분기 가이드

### 운영 시나리오

```
[강사용 분기 판단 기준]

■ 초급반 (메인 프롬프트만 사용)
  - Google AI Studio를 처음 사용하는 분
  - 프롬프트 복사-붙여넣기 후 결과 확인에 집중
  - Build 모드 기본 조작법 익히기가 목표
  - 소요 시간: 15~20분

■ 상급반 (메인 + Firecrawl 확장)
  - 메인 프롬프트를 10분 내에 완성한 분
  - API 연동 경험이 있거나 관심 있는 분
  - 외부 API 연결을 통한 확장 가능성 체험이 목표
  - 소요 시간: 25~30분
```

### 실습 타임라인

| 시간 | 초급반 | 상급반 |
|------|--------|--------|
| 0~5분 | Build 모드 진입, 모델/도구 설정 | 동일 |
| 5~10분 | 메인 프롬프트 입력 및 생성 | 동일 |
| 10~15분 | 생성된 앱 테스트, 검색어 변경 | 메인 앱 완성 확인 |
| 15~20분 | 프롬프트 수정으로 UI 개선 | Firecrawl API 키 설정 |
| 20~25분 | 결과 공유 및 토론 | 상급 프롬프트 입력 및 생성 |
| 25~30분 | - | 크롤링 기능 테스트 및 결과 공유 |

## 핵심 메시지 강조 포인트

```
[실습 마무리 시 반드시 전달]

1. "IT 부서 대기 3개월 → AI 지시 30분"
   - 방금 여러분이 만든 이 대시보드를 IT에 의뢰했다면?
   - 요구사항 정의 2주 → 개발 2개월 → QA 2주 → 총 3개월

   - 여러분이 작성한 프롬프트가 곧 요구사항 정의서
   - 구체적으로 쓸수록 결과물의 품질이 올라감
   - 이것이 바로 '바이브코딩'의 핵심

## 트러블슈팅

| 문제 상황 | 해결 방법 |
|-----------|-----------|
| Google Search 그라운딩이 안 됨 | Build 모드에서 도구 설정 → Google Search 토글 ON 확인 |
| 생성된 앱이 실행되지 않음 | "Preview" 버튼 클릭, 또는 프롬프트 끝에 "실행 가능한 단일 HTML 파일로" 추가 |
| 차트가 표시되지 않음 | Chart.js CDN 링크가 포함되었는지 확인, 프롬프트에 CDN 명시 |
| Firecrawl API 오류 | API 키 정확성 확인, 무료 크레딧 잔량 확인, CORS 이슈 시 프록시 필요 안내 |
| 한글이 깨짐 | `<meta charset="UTF-8">` 포함 여부 확인 |
