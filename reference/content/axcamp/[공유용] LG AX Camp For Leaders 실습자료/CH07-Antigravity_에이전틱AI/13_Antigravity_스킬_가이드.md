# CH07 Antigravity 스킬(Skill) 가이드

> **핵심 메시지**: "스킬 = SOP의 디지털 버전"
>
> **실습 도구**: Antigravity (https://antigravity.so)
> **소요 시간**: 약 15~20분

---

## 강사 노트

```
[진행 가이드]
- 스킬 개념 설명 5분 → 저장 실습 5분 → 재사용 실습 5분 → 챌린지 5분
- 초급반: 사전 제작 스킬 불러와서 사용하는 것에 집중
- 상급반: 직접 스킬을 만들어보는 챌린지 진행
- "SOP를 AI에 저장하는 것"이라는 비유를 반복 강조
- 스킬 = 반복 업무의 자동화 템플릿이라는 점 부각
```

## 1. 스킬이란?

### 개념 설명

```
[강의 멘트]

"여러분 회사에 SOP(표준 운영 절차)가 있죠?
월간 보고서 만들 때 항상 같은 순서로 하잖아요.

1. 데이터 수집
2. 분석
3. 차트 만들기
4. 보고서 포맷에 맞추기
5. 검토 후 제출

이 SOP를 AI에게 기억시키는 것이 '스킬'입니다.
한 번 만들어두면, 다음부터는 데이터만 바꿔 넣으면 됩니다.

SOP 문서는 서랍에 있고, 스킬은 AI 안에 있습니다.
둘 다 같은 역할인데, 스킬은 실행까지 자동으로 해줍니다."
```

### SOP vs 스킬 비교

| 구분 | 기존 SOP | Antigravity 스킬 |
|------|----------|-----------------|
| 형태 | 문서 (워드, PPT) | AI에 저장된 워크플로우 |
| 실행 | 사람이 단계별 수동 수행 | AI가 자동 실행 |
| 수정 | 문서 업데이트 후 재배포 | 스킬 편집 → 즉시 반영 |
| 공유 | 파일 공유, 교육 필요 | 스킬 공유 → 누구나 즉시 사용 |
| 일관성 | 담당자마다 품질 편차 | 항상 동일한 품질 |
| 소요 시간 | 수 시간 ~ 수 일 | 수 분 |

## 2. 스킬 저장하기

### Step-by-Step

#### Step 1: 워크플로우 완료 확인

```
[화면 설명]
Antigravity에서 프롬프트 실행이 완료된 상태.
산출물(HTML, PPT 등)이 정상적으로 생성된 것을 확인합니다.
```

- Antigravity 채팅에서 원하는 작업을 프롬프트로 실행
- 산출물이 만족스럽게 생성되었는지 확인
- 필요 시 추가 프롬프트로 수정 ("차트 색상을 빨간색으로 변경해줘" 등)

#### Step 2: 스킬로 저장

```
[화면 설명]
채팅 화면 상단 또는 워크플로우 완료 후 나타나는
"스킬로 저장(Save as Skill)" 버튼을 클릭합니다.
```

- 워크플로우 완료 후 **"Save as Skill"** (또는 "스킬로 저장") 클릭
- 또는 좌측 메뉴 → Skills → "+" 버튼

#### Step 3: 스킬 정보 입력

```
[화면 설명]
스킬 저장 다이얼로그가 나타납니다.
이름, 설명, 파라미터를 설정합니다.
```

| 입력 항목 | 예시 | 설명 |
|-----------|------|------|
| 스킬 이름 | "이사회 보고서 3종 생성" | 한눈에 용도를 알 수 있게 |
| 설명 | "매출 데이터를 분석하여 HTML 대시보드, PPT, 요약 보고서를 생성" | 상세 설명 |
| 입력 파라미터 | "분석 대상 데이터 파일" | 매번 바뀌는 입력값 정의 |
| 카테고리 | "보고서" / "분석" 등 | 정리용 태그 |

#### Step 4: 저장 완료

```
[화면 설명]
"저장" 클릭 → 좌측 Skills 메뉴에 새 스킬이 추가됩니다.
```

- "저장" 클릭
- 좌측 Skills 목록에서 저장된 스킬 확인
- 스킬 카드에 이름, 설명, 생성일이 표시됨

### 강사 팁

```
[실습 안내]
"방금 전 실습에서 만든 HTML+PPT 동시 생성 워크플로우를
스킬로 저장해 보세요. 이름은 자유롭게 지어주세요.
저장이 되면 다음부터는 데이터만 넣으면 같은 결과가 나옵니다."
```

## 3. 스킬 수정하기

### Step-by-Step

#### Step 1: 스킬 목록에서 선택

```
[화면 설명]
좌측 메뉴 → Skills → 수정할 스킬 클릭
```

- 좌측 Skills 메뉴 클릭
- 수정하려는 스킬 선택
- 스킬 상세 화면 진입

#### Step 2: 편집 모드 진입

```
[화면 설명]
스킬 상세 화면에서 "편집(Edit)" 버튼 클릭
```

- "Edit" (또는 "편집") 버튼 클릭
- 프롬프트 내용, 파라미터 등 수정 가능 상태로 전환

#### Step 3: 수정 항목

| 수정 가능 항목 | 예시 |
|---------------|------|
| 프롬프트 내용 | PPT 슬라이드 수 변경 (5장 → 7장) |
| 파라미터 | 입력 파라미터 추가/삭제 |
| 출력 형식 | HTML만 → HTML + PPT + PDF로 확장 |
| 브랜드 설정 | 컬러 코드 변경, 폰트 변경 |
| 설명 | 용도 설명 업데이트 |

#### Step 4: 저장

```
[화면 설명]
수정 후 "저장(Save)" 클릭 → 즉시 반영
```

- 수정 완료 후 "Save" 클릭
- 다음 실행부터 수정된 내용이 적용됨

### 강사 팁

```
[실습 안내]
"저장한 스킬에서 PPT의 브랜드 컬러를
여러분 회사 색상으로 바꿔보세요.
예: #A50034 → 여러분 회사 메인 컬러"
```

## 4. 스킬 재사용하기

### Step-by-Step

#### Step 1: 스킬 불러오기

```
[화면 설명]
좌측 Skills 메뉴 → 사용할 스킬 클릭 → "실행(Run)" 버튼
또는 새 채팅에서 "/" 입력 → 스킬 목록에서 선택
```

- 방법 A: Skills 메뉴에서 직접 선택 후 "Run"
- 방법 B: 새 채팅 시작 → "/" 입력 → 스킬 검색/선택

#### Step 2: 입력값 설정

```
[화면 설명]
스킬 실행 시 파라미터 입력 화면이 나타남.
새로운 데이터 파일을 업로드하거나 값을 입력합니다.
```

- 정의된 파라미터에 맞게 입력값 제공
- 예: 새로운 월간 매출 데이터 파일 업로드
- 이전 입력값이 기본값으로 표시될 수 있음

#### Step 3: 실행 및 결과 확인

```
[화면 설명]
"실행" 클릭 → AI가 스킬에 정의된 워크플로우를 자동 수행
→ 동일한 포맷의 산출물이 새로운 데이터로 생성됨
```

- 실행 클릭
- AI가 저장된 워크플로우 순서대로 자동 실행
- 결과물 확인 및 다운로드

### 다른 문서에 적용하기

```
[강의 멘트]
"1월 데이터로 만든 스킬을 2월 데이터에 적용하면?
데이터만 바꾸면 동일한 포맷의 보고서가 나옵니다.
매달 보고서 만드는 데 3시간 걸렸다면,
이제는 3분입니다."
```

| 상황 | 바꾸는 것 | 바뀌지 않는 것 |
|------|-----------|---------------|
| 1월 → 2월 보고 | 데이터 파일 | 분석 방법, 차트 종류, PPT 포맷 |
| A사업부 → B사업부 | 부문명, 데이터 | 보고서 구조, 디자인 |
| 국내 → 해외 | 데이터, 통화 단위 | 대시보드 레이아웃, KPI 구성 |

## 5. 초급반용: 사전 제작 스킬

### 스킬 A: "보고서 분석 → 3종 산출물 생성"

```
[강사 준비사항]
이 스킬은 강사가 사전에 제작하여 수강생에게 공유합니다.
수강생은 스킬을 불러와서 자신의 데이터로 실행만 하면 됩니다.
```

**스킬 설명**: 업로드된 데이터 파일을 분석하여 3가지 산출물을 자동 생성

| 산출물 | 형식 | 내용 |
|--------|------|------|
| 1. 분석 요약 | 텍스트 | 핵심 인사이트 5개, 리스크 3개, 권고사항 3개 |
| 2. 대시보드 | HTML | KPI 카드 + 차트 3개 + 데이터 테이블 |
| 3. 보고서 | PPT | 이사회 보고용 5~7 슬라이드 |

**사용 방법**:
1. Skills에서 "보고서 분석 → 3종 산출물 생성" 선택
2. 데이터 파일 업로드
3. "실행" 클릭
4. 3가지 산출물 확인 및 다운로드

### 스킬 B: "HTML+PPT 동시 출력"

```
[강사 준비사항]
12_Antigravity_HTML_PPT_프롬프트.md의 프롬프트를
사전에 스킬로 저장해둡니다.
```

**스킬 설명**: 데이터를 분석하여 인터랙티브 HTML 대시보드와 이사회 보고용 PPT를 동시 생성

| 산출물 | 형식 | 특징 |
|--------|------|------|
| 1. 대시보드 | HTML | 인터랙티브, 필터링, 차트 |
| 2. 보고서 | PPT | LG 브랜드 컬러, 이사회 포맷 |

**사용 방법**:
1. Skills에서 "HTML+PPT 동시 출력" 선택
2. 데이터 파일 업로드
3. "실행" 클릭
4. HTML 파일과 PPT 파일 각각 다운로드

### 초급반 실습 흐름

```
[타임라인]
0~3분: 스킬 개념 설명 (SOP의 디지털 버전)
3~5분: Skills 메뉴에서 사전 제작 스킬 찾기
5~8분: 스킬 A 실행 (샘플 데이터 사용)
8~12분: 결과 확인 및 다운로드
12~15분: 스킬 B로 반복 실습
15~20분: 자신의 데이터로 시도 (있는 경우)
```

## 6. 상급반용: 직접 스킬 생성 챌린지

### 챌린지: "나만의 SOP를 스킬로 만들기"

```
[강의 멘트]
"이번에는 여러분이 직접 스킬을 만들어 볼 차례입니다.
여러분이 매주, 매달 반복하는 업무가 있죠?
그 업무의 SOP를 AI 스킬로 만들어 보세요."
```

#### 챌린지 안내

**과제**: 본인의 반복 업무 중 하나를 선택하여 Antigravity 스킬로 만들기

**진행 순서**:

| 단계 | 시간 | 활동 |
|------|------|------|
| 1. 업무 선정 | 2분 | 반복 업무 중 스킬화할 대상 선택 |
| 2. SOP 정리 | 3분 | 해당 업무의 절차를 프롬프트로 작성 |
| 3. 프롬프트 실행 | 5분 | Antigravity에서 프롬프트 실행 및 결과 확인 |
| 4. 스킬 저장 | 3분 | 워크플로우를 스킬로 저장 |
| 5. 테스트 | 2분 | 다른 데이터로 스킬 재실행 테스트 |

#### 스킬 아이디어 예시

| 직무 | 반복 업무 | 스킬 이름 (예시) |
|------|-----------|-----------------|
| 경영기획 | 주간 경영 실적 정리 | "주간 경영실적 보고서 생성" |
| 마케팅 | 캠페인 성과 분석 | "캠페인 ROI 분석 리포트" |
| HR | 월간 인력 현황 보고 | "인력현황 대시보드 생성" |
| 재무 | 월말 결산 요약 | "월간 결산 요약 보고서" |
| R&D | 특허 동향 분석 | "기술/특허 트렌드 분석" |
| 영업 | 거래처 실적 정리 | "거래처 매출 분석 대시보드" |

#### 스킬 작성 템플릿

```
[수강생에게 배포]

■ 나의 스킬 설계서

1. 스킬 이름: _________________________
2. 용도: 매 [주/월/분기]마다 반복하는 [업무명]
3. 입력: [어떤 데이터/문서를 넣을 것인지]
4. 출력: [어떤 산출물이 나와야 하는지]
   - 산출물 1: __________ (형식: __________)
   - 산출물 2: __________ (형식: __________)
5. 현재 이 업무에 걸리는 시간: ___시간
6. 스킬로 만들면 예상 시간: ___분
```

#### 챌린지 평가 기준

```
[강사 평가 포인트 - 점수 아닌 피드백용]
1. 실제 반복 업무를 선정했는가? (현실성)
2. 프롬프트가 구체적인가? (구현 가능성)
3. 파라미터가 적절한가? (재사용성)
4. 산출물이 명확한가? (실용성)
```

| 시간 | 초급반 | 상급반 |
|------|--------|--------|
| 0~5분 | 스킬 개념 설명 | 스킬 개념 설명 |
| 5~10분 | 사전 스킬 불러오기 및 실행 | 사전 스킬 실행 (빠르게) |
| 10~15분 | 결과 확인, 다른 스킬 시도 | 직접 스킬 생성 시작 |
| 15~20분 | 자유 실습 / Q&A | 스킬 저장 및 테스트 |

## 핵심 메시지 정리

```
[실습 마무리 시 반드시 전달]

1. "스킬 = SOP의 디지털 버전"
   - 종이 SOP: 절차만 적혀있고, 사람이 실행
   - AI 스킬: 절차가 저장되어 있고, AI가 실행

2. "한 번 만들면 계속 쓴다"
   - 이번 달에 3시간 걸린 보고서
   - 스킬 만드는 데 30분
   - 다음 달부터? 3분
   - 1년이면 30시간 이상 절약

3. "공유하면 팀 전체가 쓴다"
   - 나만의 노하우가 담긴 SOP를 팀에 공유
   - 신입사원도 베테랑과 동일한 품질의 산출물
   - 조직 역량의 표준화

4. "SOP를 만들 줄 아는 사람이 스킬도 만든다"
   - 업무를 잘 아는 사람이 최고의 스킬을 만듦
   - AI가 대체하는 것이 아니라, 전문가의 노하우를 증폭시키는 것
```

## 트러블슈팅

| 문제 상황 | 해결 방법 |
|-----------|-----------|
| Skills 메뉴가 보이지 않음 | Antigravity 좌측 사이드바 확인, 로그인 상태 확인 |
