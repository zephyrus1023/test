기술 활용 로드맵

오늘 수업은 챗봇에서 시작해 바이브코딩을 거쳐 Agent까지 올라갑니다. 아래에는 NotebookLM 노트북에서 내려받은 실제 슬라이드 3장을 배치해, 각 단계의 인터페이스와 실습 도구를 한 번에 연결해 보여줍니다.

NotebookLM Slide Deck

대화형 어시스턴트에서 실행형 에이전트로 확장되는 3단계 활용 로드맵

NotebookLM 스튜디오 슬라이드 Enterprise AI Roadmap (2)를 PDF로 내려받아 그대로 붙였다. 챗봇, 바아브코딩, Agent 각 단계에서 어떤 인터페이스와 도구를 쓰는지, 오늘 실습 동선과 함께 본다.

다운로드

Level 1 · 입문
어시스턴트

질문하고, 리서치하고, 보고서 초안을 받고, 다시 수정시키는 가장 빠른 협업 루프다. 사람이 목표와 맥락을 잡고 AI가 반복 속도를 높이는 방식입니다.

- Gemini로 아이디어 탐색, 요약, 초안 작성

- NotebookLM으로 source-grounded research와 근거 확인

- 보고서 초안, 회의 정리, 브레인스토밍처럼 짧은 왕복이 많은 작업에 적합

Gemini
NotebookLM
ChatGPT

Level 2 · 확장
에이전트

AI의 도움으로 앱을 더 빨리 만들고, 완성된 앱이 다시 LLM API를 호출해 맞춤형 서비스를 제공하는 단계다. 개발 생산성과 서비스 자동화가 같이 올라갑니다.

- AI Studio Build로 서비스 흐름과 화면 초안을 빠르게 설계

- Gemini API로 앱 안에 요약, 분석, 생성 기능을 내장

- 사내 데이터와 외부 API를 묶어 특정 업무에 맞는 전용 서비스를 구현

AI Studio Build

Level 3 · 자율
Agentic AI

터미널을 인터페이스로 삼아 AI가 계획을 세우고, 도구를 호출하고, 실행 결과를 보며 다시 수정하는 단계다. 오늘은 Hi-D Code를 활용하여 이 흐름을 시연합니다.

- 문제를 작은 작업으로 나누고 순서대로 실행하는 harness 관점

- 코드, 셸, 브라우저, 테스트를 엮어 자율적 workflow를 구성

- 복잡한 문제 해결, 반복 수정, 검증 루프가 많은 작업에 적합

Hi-D Code
Cline

오늘의 여정

오전에는 Gemini와 ChatGPT, NotebookLM을 통해 질문, 리서치, 보고서 초안 루프를 경험합니다. 이어서 Gems, Google AI Studio로 바이브코딩을 통한 앱 제작을 다루고, 마지막에는 Agentic AI를 Hi-D Code(Cline) 활용한 agentic workflow를 통해 어떻게 구현할 수 있는지 직접 시연합니다.
