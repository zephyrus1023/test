~35분 CH 04 실습

프롬프팅 기초

처음부터 무거운 업무 프롬프트로 들어가기보다, Gemini와 자연스럽게 대화하며 페르소나 설정, 맥락 누적, 구조화된 정리를 몸으로 익히는 클립입니다.
이번 실습의 흐름
1. Warm Intro
질문지처럼 쓰지 않고, 편한 자기소개로 시작합니다. Gemini는 그 소개를 받아 AI 파트너처럼 대화를 이어갑니다.
2. Build-up
추가 질문을 통해 경력, 강점, 관심 분야, 교육 목표, AI로 풀고 싶은 문제를 점점 더 선명하게 만듭니다.
3. Shareable Profile
대화 결과를 공유용 소개, 상세 프로필, Markdown 저장본으로 정리해 다시 쓰기 좋은 자산으로 만듭니다.
4. Profile Page
마지막에는 프로필을 웹페이지처럼 꾸며보며, 소개 내용을 보여주고 공유하는 형태로 확장합니다.
이번 클립의 도착점
이 대화가 끝나면 참가자는 공유용 자기소개, 상세 업무 프로필, Markdown 저장본, 웹페이지 초안까지 한 번에 가져가게 됩니다.
1. 먼저 이렇게 시작해보세요

처음부터 빈칸을 채우는 대신, 편하게 자기소개를 시작하고 Gemini가 자연스럽게 인터뷰를 이어가게 하는 편이 훨씬 좋습니다. 중요한 것은 말투를 딱딱하게 맞추는 것이 아니라, 나의 경력과 기대, 문제의식을 점점 더 잘 드러내는 것입니다.

자기소개 대화 시작 템플릿
당신은 이번 AX Camp에서 내 자기소개를 함께 다듬어주는 AI 파트너입니다. 내가 편하게 자기소개를 시작하면, 경력과 강점, 관심 분야, 이번 교육에서 얻고 싶은 것, AI로 풀고 싶은 업무 문제를 더 잘 드러낼 수 있도록 질문을 이어가 주세요. 조건: - 질문은 한 번에 1~2개씩만 - 말투는 자연스럽고 편안하게 - 빠진 정보는 추정하지 말고 질문으로 보완 - 중간중간 "현재까지 정리된 내 소개"를 3줄로 업데이트 마지막에는 아래 3가지를 만들어 주세요. 1. 공유용 짧은 소개 2. 상세 프로필 3. markdown 코드블록 버전 먼저 내가 편하게 자기소개를 시작할게. 자연스럽게 이어가 주세요.
2. Gemini에게 추가 질문을 부탁해보세요

한 번 답하고 끝내지 말고, Gemini가 나를 더 잘 이해하도록 질문을 이어가게 만드는 것이 중요합니다. 경력, 강점, 관심 분야, 이번 교육 목표, AI 과제까지 골고루 드러나야 나중에 프로필의 완성도가 높아집니다.

추가 질문 요청
지금까지 들은 내용을 바탕으로, 내 소개가 더 입체적으로 드러나도록 질문을 이어가줘. 특히 아래 영역이 균형 있게 드러나도록 해줘. - 현재 역할과 책임 - 경력에서 강점이 드러나는 경험 - 관심 산업/주제 - 이번 교육에서 기대하는 변화 - AI로 해결하고 싶은 실제 업무 문제 조건: - 한 번에 1~2개씩만 질문 - 왜 이 질문이 필요한지도 한 줄로 덧붙이기 - 내 답변을 반영해서 다음 질문을 조정하기
프로필 보강 요청
지금까지 내용을 기준으로, 내 소개에서 아직 약하게 보이는 부분만 골라서 보강해줘. 아래 관점으로 아직 부족한 정보를 짚어주고, 필요한 것만 추가 질문해줘. - 현재 맡고 있는 일 - 잘 아는 산업/도메인 - 남들이 나를 기억할 만한 강점 - 이번 교육에서 꼭 얻어가고 싶은 것 - AI를 통해 바꾸고 싶은 업무 방식 - 수업 후 다른 사람과 공유해도 좋을 자기소개 포인트
3. 대화 내용을 구조화해 보세요

여기서부터 구조화된 정리의 가치가 드러납니다. 대화로 쌓은 내용을 일정한 형식으로 정리하면, 저장하기도 좋고 다른 사람과 공유하기도 좋고, 다음 AI 도구의 입력으로도 바로 쓸 수 있습니다.

프로필 요약 요청
지금까지 대화를 바탕으로 내 프로필을 아래 형식으로 정리해줘. ## 한 줄 소개 ## 현재 역할과 책임 ## 경력 하이라이트 ## 관심 분야와 강점 ## 이번 교육에서 얻고 싶은 것 ## AI로 풀고 싶은 업무 문제 ## 같이 대화해보고 싶은 주제 마지막에는 아래도 함께 붙여줘. - 동료들과 바로 공유할 수 있는 3문장 소개 - Gemini에게 다음으로 해보면 좋은 질문 3개
Markdown 코드블록으로 저장하기
방금 정리한 프로필을 다시 한 번 정돈해서 출력해줘. 조건: - 전체 내용을 ```markdown 코드블록``` 안에 넣어줘. - 제목과 소제목, 불릿을 유지해줘. - 발표 전에 내가 다시 읽기 쉽도록 너무 길지 않게 정리해줘. - 나중에 다른 AI 도구나 문서에 바로 붙여넣을 수 있게 깔끔하게 정리해줘.
왜 이 형식이 중요한가
AI는 긴 자유서술도 이해하지만, 제목·불릿·코드블록처럼 경계가 분명한 형식을 훨씬 안정적으로 다룹니다. 그래서 Markdown으로 정리된 결과는 저장, 공유, 재입력에 모두 유리하고, 다음 프롬프트의 맥락으로도 바로 이어 쓰기 좋습니다.
4. 프로필을 웹페이지처럼 꾸며 공유해보세요

정리된 프로필은 저장만 하고 끝내지 말고, 한 장짜리 소개 페이지처럼 꾸며보면 훨씬 재미있습니다. 수업 중 서로 보여주며 자기소개를 나눌 수도 있고, AI가 정보를 시각적으로 재구성하는 방식도 함께 체감할 수 있습니다.

프로필 기반 웹페이지 생성 요청
방금 정리한 내 프로필을 바탕으로, 다른 사람들과 공유할 수 있는 한 페이지짜리 자기소개 웹페이지를 만들어줘. 조건: - 첫 화면에서 내가 어떤 사람인지 3초 안에 이해되게 - 섹션: 한 줄 소개 / 경력 하이라이트 / 관심 분야 / 이번 교육에서 얻고 싶은 것 / AI로 풀고 싶은 업무 문제 - 톤: 깔끔하고 전문적인 스타일 - 너무 화려하지 않게 - 발표 중 잠깐 보여줘도 어색하지 않게 - 가능하면 공유용 소개 카드처럼 보이게 구성해줘 - 먼저 전체 구성안을 보여주고, 이어서 실제 페이지 초안을 제안해줘.
