# Gems 소개: AI 비서 만들기

앞에서 익힌 **페르소나 설정**, **맥락 누적**, **구조화된 출력**을 이제 실제 업무 자료에 적용합니다. 회의 오디오와 참고 문서를 분석하고, 그 흐름을 반복 가능한 **AI 비서**로 고정하는 클립입니다.

NotebookLM Infographic

## 비즈니스 프롬프트를 업무 위임 구조로 바꾸는 법

오디오 회의록 작성, 핵심 이슈 추출, 자료 교차검증, CEO 브리프 작성까지 하나의 워크플로로 정리한 인포그래픽입니다. 이 흐름을 그대로 AI 비서에 저장하는 것이 이번 클립의 핵심입니다.

다운로드

## 이번 클립의 흐름

1. Input

회의 오디오와 맥락 문서를 같은 대화창에 붙여, AI가 참고할 재료를 먼저 모읍니다.

2. Analyze

회의록 작성, 핵심 이슈 추출, 데이터 교차검증으로 분석의 깊이를 단계적으로 올립니다.

3. Brief

최종적으로 CEO용 의사결정 브리프와 후속 Deep Research 질문까지 이어서 만듭니다.

4. Reuse

잘 된 흐름은 Gemini Gems에 저장해, 반복되는 회의 분석과 브리핑 작성 업무를 비서처럼 재사용합니다.

## 실습 준비파일

버튼을 누르면 실습용 구글 드라이브 링크가 클립보드에 복사됩니다. 같은 대화창에 순서대로 붙여서 실습하세요.

회의 오디오
맥락 문서

## 회의 오디오로 실습하는 4단계

![Gemini 파일 업로드 메뉴](/assets/gemini/ch02/gemini-upload-menu.png)

회의 녹음과 맥락 자료는 모두 입력창 왼쪽의 파일 업로드 메뉴에서 붙입니다. 핵심은 **같은 대화창을 유지하면서 단계별로 깊이를 더하는 것**입니다.

| 단계 | 입력 | AI에게 시키는 일 | 산출물 |
| --- | --- | --- | --- |
| **Step 1** | 회의 오디오 | 구조화된 회의록 작성 | 참석자, 발언 요지, 의사결정, 미해결 이슈 |
| **Step 2** | 방금 만든 회의록 | 전략 이슈와 의사결정 갭 추출 | 핵심 이슈 목록, 리스크, 다음 회의 안건 |
| **Step 3** | 회의 맥락 참고자료 | 회의 내용과 데이터 교차검증 | 불일치 포인트, 놓친 데이터, 재검토 항목 |
| **Step 4** | 앞선 모든 분석 결과 | CEO용 의사결정 브리프 작성 | 핵심 요약, 옵션 비교, 추천 액션 |
## 바로 써보는 프롬프트 5개

1. 음성 기반 AI 회의록 작성복사

이 음성을 듣고 아래 형식으로 구조화된 회의록을 작성해 주세요.

- 회의명, 일시, 참석자(직책 포함)

### 2. 안건별 논의 내용
- 안건 제목
- 발언 요약
- 쟁점

### 3. 결정사항
- 번호 매기기, 담당자 명시

### 4. 후속조치(Action Items)
| 번호 | 조치 내용 | 담당자 | 기한 |
| --- | --- | --- | --- |
|------|----------|--------|------|
### 5. 미결 사항
- 추가 논의가 필요한 이슈

2. 회의록 기반 핵심 이슈 추출복사

방금 작성한 회의록을 기반으로, 경영 전략 애널리스트 관점에서 분석해 주세요.

## 요청 사항
### 1. 핵심 전략 이슈 TOP 3
- 이슈명
- 배경
- 회의에서의 논의 수준
- 리스크 수준

### 2. 의사결정 갭 분석
- 결정은 됐지만 실행 방안이 불명확한 항목
- 논의는 됐지만 결정이 안 된 항목
- 아예 다뤄지지 않았지만 다뤄졌어야 할 항목

### 3. 다음 회의 어젠다 추천
- 반드시 다뤄야 할 안건 2~3개
- 예상 소요 시간과 필요 참석자

3. 참고자료 첨부 + 멀티턴 분석복사

이 참고자료와 앞서 정리한 회의록을 교차 분석해서 다음을 정리해 주세요.

## 요청 사항
### 1. 회의 논의 vs 실제 데이터 대조표
| 회의에서 언급된 내용 | 참고자료의 실제 데이터 | 일치 여부 | 비고 |

|---------------------|----------------------|----------|------|
### 2. 계획 대비 실적 갭
### 3. 회의에서 놓친 포인트
### 4. 데이터 기반 수정 권고

4. CEO 의사결정 브리프 작성복사

지금까지의 분석(회의록, 핵심 이슈, 참고자료 교차분석)을 종합하여 CEO의 의사결정을 돕는 전략 보좌관 역할로 최종 권고안을 작성해 주세요.

## 출력 형식
### 1. 상황 종합 (3문장)
### 2. 전략 옵션 비교표
### 3. 추천안 및 근거
### 4. 실행 로드맵
### 5. 모니터링 KPI

5. Deep Research로 확장하기복사

앞선 분석에서 가장 중요한 전략 이슈 하나를 골라, Deep Research용 질문으로 다시 써 주세요.

조건:
- 조사 목적을 한 줄로 명확히 적기
- 비교해야 할 경쟁사 또는 외부 사례 포함
- 필요한 데이터와 출처 기준 제시
- 마지막에는 '경영진 시사점' 섹션이 들어가게 만들기

## 이 흐름을 AI 비서로 저장하기

방금까지는 같은 대화창에서 회의 오디오를 분석했습니다. 이제 이 흐름을 반복 가능한 **Gem**으로 고정하면, 다음번에는 파일과 질문만 바꿔도 비슷한 품질의 회의록·브리프를 더 빠르게 재사용할 수 있습니다.

| 방금 채팅에서 한 일 | Gem에 저장할 것 | 다음번에 얻는 이점 |
| --- | --- | --- |
| 회의록 작성 방식 지정 | 회의 비서 역할, 기본 출력 템플릿 | 매번 같은 형식으로 회의록이 나옵니다 |
| 핵심 이슈와 액션 추출 | 분석 관점, 후속 질문 규칙 | 빠뜨리는 포인트가 줄어듭니다 |
| Deep Research 질문으로 확장 | 리서치 전환용 프롬프트 구조 | 심층 조사와 보고서 작성으로 자연스럽게 이어집니다 |
회의록 자동화 Gem 인스트럭션복사

당신은 기업 임원회의 회의록 자동화 Gem입니다.

사용자가 회의 오디오, 회의록 초안, 참석자 정보, 맥락 문서를 넣으면 아래 순서로 동작하세요.

1. 먼저 입력된 자료를 확인하고, 회의명·참석자·회의 목적처럼 빠진 정보가 있으면 최대 3개까지만 질문합니다.
2. 정보가 충분하면 아래 형식으로 구조화된 회의록을 작성합니다.

## 1. 회의 기본 정보
- 회의명
- 일시
- 참석자(직책 포함)

## 2. 안건별 논의 내용
- 안건 제목
- 발언 요약
- 핵심 쟁점

## 3. 결정사항
- 번호 매기기
- 담당자와 기한 표시

## 4. 후속조치(Action Items)
| 번호 | 조치 내용 | 담당자 | 기한 |

## 5. 미결 사항
- 추가 논의가 필요한 이슈

3. 회의록 작성 후 아래 항목도 함께 덧붙입니다.
- 핵심 전략 이슈 TOP 3
- 다음 회의에서 확인할 질문 3개
- Deep Research로 넘기면 좋은 조사 주제 2개

응답 원칙:
- 경영진 보고용 격식체
- 확인되지 않은 내용은 추정하지 말고 '확인 필요'로 표시
- 표와 불릿 우선
- 발언 요약은 중립적으로 작성

심층 리서치 보조 Gem 인스트럭션복사

당신은 경영진용 심층 리서치 준비 Gem입니다.

사용자가 조사 주제, 독자, 의사결정 목적, 현재 가진 맥락(회의록·내부 메모·기사 링크)을 주면 아래를 순서대로 만듭니다.

## 1. 조사 목적
- 왜 이 조사를 하는지 1문장

## 2. 핵심 질문 5개
- 의사결정에 직접 필요한 질문 위주

## 3. 비교 대상
- 경쟁사, 벤치마크 기업, 참고 사례 3~5개

## 4. 필요한 데이터와 출처 기준
- 어떤 수치가 필요한지
- 어떤 출처를 우선해야 하는지

## 5. Deep Research용 최종 질문
- 바로 복사해 넣을 수 있는 완성형 질문

## 6. 결과 정리용 보고서 목차
- 리서치 결과를 받았을 때 정리할 섹션 초안

응답 원칙:
- 최신 데이터 우선
- 질문 범위가 너무 넓으면 먼저 산업·지역·기간을 좁히기
- 경영진 시사점이 나오도록 질문을 설계
- 사용자가 넣은 회의 맥락을 반드시 반영

### 운영 포인트

이 두 Gem을 나눠 만들어 두면, **회의 → 요약 → 후속조사** 흐름을 파일과 주제만 바꿔 반복 실행할 수 있습니다. 즉, 잘 된 실습을 일회성 대화가 아니라 재사용 가능한 업무 체계로 바꾸는 것입니다.

## 실습 뒤에 바로 해보는 테스트 질문

테스트 1: 임원 브리핑 1페이지복사

방금 작성한 회의록과 핵심 이슈를 바탕으로, CEO에게 보고할 1페이지 브리프를 다시 정리해줘.

조건:
- 상반기 주요 성과 3개
- 지금 가장 시급한 리스크 2개
- 이번 회의에서 바로 판단해야 할 결정사항 3개
- 마지막에 권고 우선순위 1, 2, 3을 적어줘

테스트 2: 맥락 문서까지 반영한 재검토복사

회의록과 맥락 문서를 함께 반영해서, 아래 세 축으로 다시 검토해줘.

1. EXAONE 도입률 목표 미달의 원인과 병목
2. AI/ML 인력 충원 병목과 전환 교육 대안
3. 외부 LLM 사용 비율 31%가 가져오는 비용·보안 리스크

출력:
- 현재 상황 요약
- 놓친 데이터 또는 추가 확인 필요사항
- 다음 회의 전 액션 아이템

테스트 3: Deep Research로 연결복사

지금까지 나온 이슈 중 가장 중요한 것 하나를 골라, Deep Research용 질문과 보고서 목차 초안을 만들어줘.

우선순위 후보:
- 하반기 AX 투자 6,300억 원의 우선순위 조정
- AI 인력 재배치와 전환 교육 확대
- 경쟁사 대비 LG의 차별화 포인트 정립

조건:
- 조사 목적 1문장
- 꼭 비교해야 할 경쟁사 또는 외부 사례 3개
- 필요한 데이터 항목과 출처 유형
- 최종 보고서 목차 5개

### 핵심 메시지

좋은 AI 비서는 마법처럼 모든 것을 아는 캐릭터가 아니라, **잘 정의된 역할 + 일관된 형식 + 반복 가능한 컨텍스트**의 묶음입니다. 방금 잘 됐던 프롬프트 흐름을 Gem으로 저장하면, 같은 유형의 회의 분석과 브리핑 초안을 훨씬 빠르게 재사용할 수 있습니다.
