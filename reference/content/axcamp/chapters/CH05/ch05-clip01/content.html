LLM 앱과 API의 동작 원리

AI Studio Build로 만든 앱을 로컬에서 실행해도 실제 추론은 원격 Gemini API에서 일어납니다. 이 섹션은 API 방식, key, quota, billing이 왜 필요한지를 한 번에 이해하시도록 정리한 개념 오리엔테이션입니다.

NotebookLM Slide Deck

AI Studio 앱이 왜 API 키와 원격 추론에 의존하는지 6장으로 이해하기

이번 챕터 설명을 바탕으로 NotebookLM에서 만든 실제 슬라이드다. 로컬 앱과 Gemini API의 연결, API key의 역할, quota와 비용, 배포 전 설정 포인트를 한 번에 훑을 수 있다.

다운로드
왜 이 챕터가 필요한가
1. Chat UI
Gemini나 NotebookLM에서는 로그인만 하면 바로 쓰기 때문에 모델 호출 구조와 과금 기준이 화면 뒤로 숨어 있다.
2. API / Build
AI Studio Build로 앱을 만들기 시작하면, 이제 앱이 직접 Gemini API를 호출해야 하므로 key와 quota, billing 개념이 전면으로 나옵니다.
3. Local Run
내려받은 앱을 로컬에서 실행해도 LLM이 PC 안에 들어온 것은 아니다. 앱은 여전히 원격 모델 서비스를 부르는 클라이언트다.
오늘 수업 연결
오늘은 AI Studio로 만든 앱이 어떻게 동작하는지 이해하는 것이 핵심입니다. API 키를 세세하게 다루기보다, 왜 필요한지와 어디에 연결되는지를 먼저 잡아두면 이후 실습이 훨씬 쉬워진다.
실제 AI Studio 화면으로 보기
Playground에서는 모델, 시스템 지시문, 도구, API key 상태를 한 화면에서 확인할 수 있습니다. 즉, Chat UI와 API 실행 설정이 만나는 작업대다.
Build는 자연어로 앱을 설계하고 내려받는 공간입니다. 하지만 앱을 배포하거나 로컬에서 실행하려면 결국 Gemini API에 연결할 인증 정보가 필요합니다.
실행 전에 이해할 4가지
개념무엇을 뜻하나실습에서의 의미
API Key누가 어떤 프로젝트로 모델을 호출하는지 식별하는 출입증나중에 Build 앱을 로컬에서 실행할 때 반드시 연결해야 합니다.
Quota / Rate Limit얼마나 자주, 얼마나 많이 호출할 수 있는지에 대한 사용량 제한Chat UI보다 API 앱에서 더 직접적으로 체감된다.
Billing토큰 처리, 추론 계산, grounding과 도구 사용에 따른 원격 연산 비용무료 체험 범위 안에서는 감춰져 있어도, 원리는 똑같이 작동합니다.
Environment Variable앱 코드에 비밀값을 직접 넣지 않고 `GEMINI_API_KEY` 같은 변수로 분리하는 방식보안과 배포를 위해 기본으로 익혀야 하는 운영 습관입니다.
