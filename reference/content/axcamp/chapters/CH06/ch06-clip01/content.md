## 왜 지금 Vibe Coding인가

1. 속도

기획서를 쓰는 시간과 앱 초안이 나오는 시간이 거의 같은 흐름으로 붙는다. 아이디어를 설명하면 바로 형태가 나옵니다.

2. 반복

초안 생성 뒤 곧바로 “이 카드 더 크게”, “이 표를 더 임원 친화적으로” 같은 피드백을 주며 한 대화 안에서 개선할 수 있습니다.

3. 역할 분담

AI가 코드를 더 많이 쓰더라도, 문제 정의와 품질 판단은 사람이 쥔다. 리더는 “무엇을 만들지”와 “어떻게 고칠지”에 집중합니다.

## 기존 개발과 무엇이 다른가

| | 기존 방식 | Vibe Coding |
| --- | --- | --- |
| **출발점** | 요구사항 문서 → 개발 요청 → 구현 | 자연어 지시 → AI가 초안 생성 → 즉시 피드백 |
| **핵심 역량** | 언어 문법과 구현 세부지식 | 목표 정의, 맥락 제공, 검수와 우선순위 판단 |
| **시간 감각** | 수주~수개월 | 수분~수시간 안에 프로토타입 |
| **대표 산출물** | 정식 서비스, 대규모 시스템 | 업무용 도구, 보고용 앱, 시범 대시보드 |
