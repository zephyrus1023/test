개요
시연: 3종 보고서 → 대시보드
강사가 Antigravity를 사용하여 전략보고서에서 3종 보고서 + HTML 대시보드 + PPT를 자동 생성하는 과정을 시연합니다. AI의 자율 실행 과정에 주목하며 관람하세요.
시연 개요
• 전략보고서를 입력하여 본인용/상사 보고용/부하 지시용 3종 보고서를 동시에 생성합니다
• 같은 분석 결과를 HTML 대시보드 + PPT로 동시 출력합니다
• AI가 스스로 계획하고, 도구를 선택하고, 결과를 검증하는 에이전트 루프를 관찰합니다
3종 보고서란?

같은 사실을 수신자에 따라 다르게 가공하는 것은 임원의 핵심 역량입니다. AI는 이 정보 변환을 자동화합니다.

보고서 유형	수신자	목적	핵심 특성
본인용(自用)	본인	의사결정 참고	솔직한 판단, 리스크 우선
상사 보고용	상사 (부사장)	승인/지원 요청	격식체, 숫자 중심, 1페이지
부하 지시용	각 팀장/팀원	실행 지시	구체적 액션 아이템, 기한 명시

이 세 가지를 수작업으로 만들면 최소 2~3시간, 여기에 대시보드와 PPT까지 합하면 6~8시간이 소요됩니다. AI에게 맡기면 15~20분입니다.

시연 흐름 — 관람 가이드
단계	강사 행동	관찰 포인트
1	조직정보 + 전략보고서 업로드	AI에게 컨텍스트(맥락)를 제공하는 단계입니다. 조직정보가 상세할수록 결과 품질이 높아집니다.
2	3종 보고서 생성 프롬프트 입력	에이전트 루프 관찰: AI가 자동으로 계획을 수립하고, 3종 보고서를 동시에 작성하는 과정을 살펴보세요. Plan → Execute → Check 사이클이 반복됩니다.
3	HTML + PPT 동시 생성 프롬프트 입력	멀티에이전트 관찰: 하나의 에이전트가 대시보드를, 다른 에이전트가 PPT를 동시에 생성합니다. Manager View에서 진행 상황을 확인합니다.
4	생성된 결과물 리뷰	같은 데이터가 수신자별로 어떻게 다르게 변환되었는지, HTML과 PPT의 용도 차이를 비교합니다.

관람 포인트
시연을 관람할 때 다음 세 가지에 주목하세요:

1. 자율 계획 수립: 프롬프트를 입력한 후, AI가 스스로 작업 계획을 세우는 과정을 관찰하세요. 사람이 단계별로 지시하지 않아도 AI가 최적의 순서를 결정합니다.

2. 도구 자동 선택: AI가 필요에 따라 Editor, Terminal, Browser를 오가며 작업하는 것을 확인하세요. 코드를 작성하고, 실행하고, 결과를 확인하는 전 과정이 자동입니다.

3. 결과 자체 검증: AI가 생성한 결과물을 스스로 확인하고, 문제가 있으면 수정하는 과정을 살펴보세요. 이것이 에이전트 루프의 Check 단계입니다.
