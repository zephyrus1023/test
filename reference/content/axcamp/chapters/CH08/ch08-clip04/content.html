## 1. Executive Summary

2026년 상반기 글로벌 AI 가전 시장은 전년 대비 34.2% 성장하며 본격적인 확산기에 진입하였다. LG전자는 자체 AI 모델 EXAONE 3.5를 가전 제품군에 탑재하여 **"AI 가전 생태계"** 구축을 가속화하고, 2026 하반기 두 개의 전략 프로젝트(Project Aurora, Project Nebula)를 통해 프리미엄 시장 점유율 확대와 구독 매출 기반 확보를 동시에 추진합니다.

**핵심 목표:**
- 2026 하반기 AI 가전 매출 **8.7조원** 달성 (전사 매출의 22.3%)
- 프리미엄 AI 가전 시장 글로벌 점유율 **28.5%** 확보 (현재 23.1%)
- EXAONE 기반 구독 서비스 MAU **1,200만 명** 돌파
- AI 가전 영업이익률 **9.8%** 달성 (현재 7.2%)

## 3. 패스트 리서치 프롬프트

> **소요 시간**: 약 4분

3. 아래 리서치 주제 입력
4. NotebookLM이 자동으로 관련 웹 소스를 수집하여 노트북에 추가

### 패스트 리서치 주제

```
2025-2026년 글로벌 AI 가전 시장 트렌드와 주요 기업(LG, 삼성, 하이센스)의
AI 전략 비교 분석
```

### 대안 주제 (시간/상황에 따라 선택)

```
한국 대기업의 AI 인재 양성 전략과 리스킬링 프로그램 최신 동향
```

### 강사 노트

```
[타이밍] Fast Research가 소스를 수집하는 데 1~2분 소요.

[조작 포인트]
- Fast Research가 여러 웹 소스를 자동으로 찾아 추가하는 과정을 보여줄 것
- 수집된 소스 목록이 나오면 어떤 출처들이 포함됐는지 확인

[핵심 교수 포인트]

   → "기존에는 소스를 하나씩 직접 업로드했습니다.
      Fast Research는 AI가 주제에 맞는 웹 자료를 자동으로 찾아서
      노트북에 추가해 줍니다. 리서치 자동화입니다."

2. 기존 소스와의 통합
   → "이미 업로드한 WEF/딜로이트 보고서 + 방금 수집된 웹 소스가

   → "AI가 찾아온 소스가 신뢰할 만한지는 여러분이 판단해야 합니다.
      출처를 반드시 확인하세요."

## 4. 노트 → 소스 전환 가이드

> **소요 시간**: 약 4분
> **목적**: 대화 내용을 노트로 저장하고, 그 노트를 다시 소스로 전환하는 고급 기능 시연

### Step 4-1: 대화를 노트로 저장

**조작 순서:**
1. 앞서 진행한 질의응답 중 가장 유용했던 답변을 선택
2. 해당 답변 우측의 **"노트에 저장"** (핀 아이콘 또는 저장 버튼) 클릭
3. 노트 패널에서 내용 확인

**설명 프롬프트 (강사가 구두로 설명):**

```
"지금 AI가 정리해 준 '임원 시사점 3가지'가 유용하니까,
이걸 노트로 저장하겠습니다.
→ [답변 옆 저장 버튼 클릭]
→ 노트 패널에 저장된 것 확인"
```

### Step 4-2: 노트를 소스로 전환

**조작 순서:**
1. 좌측 노트 패널에서 저장된 노트 선택
2. 노트 상단 또는 우클릭 메뉴에서 **"소스로 전환"** 클릭
3. 해당 노트가 소스 목록에 추가된 것 확인

**전환 후 활용 프롬프트:**

```
방금 소스로 전환된 '임원 시사점 분석' 노트와
원래 WEF/딜로이트 보고서를 함께 참조해서,

각 시사점별 실행 계획을 구체화해 줘.
특히 LG그룹 임원이 다음 분기에 바로 착수할 수 있는
액션 아이템 중심으로 정리해 줘.
```

### 강사 노트

```
[타이밍]
- Step 4-1 (노트 저장): 1분
- Step 4-2 (소스 전환): 1분
- 전환 후 활용 프롬프트: 2분

[핵심 교수 포인트 - 이 기능이 NotebookLM의 킬러 피처]
1. 지식의 누적 구조
   → "AI와의 대화 결과를 노트로 저장 → 소스로 전환하면,
      이전 분석 결과가 새로운 분석의 기반이 됩니다.
      지식이 쌓이는 겁니다."

2. 실무 적용 시나리오
   → "매주 회의 후 분석 결과를 노트로 저장 → 소스로 전환
      → 다음 회의 때 이전 분석을 참조하며 새로운 분석 수행
      → 분기별 전략 리뷰에서 누적된 인사이트 종합"

3. 일반 챗봇과의 결정적 차이
   → "ChatGPT나 Gemini에서는 대화가 끝나면 지식이 사라집니다.
      NotebookLM은 노트 → 소스 전환으로 지식을 영구 보관할 수 있습니다.
      이게 '개인 지식 베이스'입니다."
```

## 추가 기능: Firecrawl 직접 크롤링

### 1. URL 직접 크롤링
- "URL 크롤링" 탭 추가
- 사용자가 특정 URL을 입력하면 Firecrawl API로 해당 페이지 내용 추출
- 추출된 내용을 AI가 자동 요약
- 지원 URL 예시:
  - 경쟁사 IR 페이지
  - 기술 블로그
  - 보도자료 페이지
  - 산업 리포트 페이지

### 2. 사이트맵 크롤링
- 특정 도메인의 최근 게시글 목록 자동 수집
- 예: "https://newsroom.samsung.com" 입력 → 최신 보도자료 10건 자동 수집
- 수집된 내용 자동 요약 및 테이블 정리

### 3. 정기 모니터링 설정 (UI만 구현)
- 모니터링할 URL 목록 관리
- 크롤링 주기 설정 UI (실제 스케줄링은 백엔드 필요하므로 UI만)
- "지금 실행" 버튼으로 수동 크롤링

### Firecrawl API 설정
- API 키 입력 필드 (설정 패널)
- API 키는 로컬스토리지에 저장
- 연결 테스트 버튼

### API 호출 예시 (참고용)
```javascript
// Firecrawl API 호출
const response = await fetch('https://api.firecrawl.dev/v1/scrape', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_API_KEY'
  },
  body: JSON.stringify({
    url: '크롤링할_URL',
    formats: ['markdown']
  })
});
```

### UI 변경사항
- 상단 탭 추가: [Google 검색] [URL 크롤링] [모니터링]

### 보고서 1: 나용(自用) — 의사결정 메모

**목적**: 이준혁 전무 본인이 참고할 핵심 요약 + 의사결정 포인트

**형식 규격**:
- 분량: A4 1~2페이지
- 문체: 반말/메모체 (예: "~해야 함", "~가 문제", "~확인 필요")
- 구조:
  1. **한줄 요약** — 현재 상황을 한 문장으로
  2. **잘 되고 있는 것** (3개 이내, 근거 수치 포함)
  3. **걱정되는 것** (3개 이내, 리스크 수준 표시)
  4. **이번 주 내가 직접 할 것** (구체적 행동 3개)
  5. **판단이 필요한 의사결정** (선택지 A/B 형태로 정리)
     - 각 선택지의 장단점과 추천 의견 포함
  6. **다음 보고까지 모니터링할 숫자** (KPI 3~5개)

**톤**: 솔직하고 냉정하게. 좋은 소식도 나쁜 소식도 있는 그대로.
외부에 보여줄 문서가 아니므로 정치적 표현 불필요.

---

### 보고서 2: 상사보고용 — 부사장 보고서

**목적**: 조현수 부사장에게 올리는 분기 성과 보고 + 4분기 전략 승인 요청

**형식 규격**:
- 분량: **정확히 1페이지** (넘기지 말 것)
- 문체: 격식체 (~하였습니다, ~입니다)
- 구조:
  1. **보고 제목** — 한 줄
  2. **3분기 성과 요약** — 핵심 수치 3개 + 한 줄 해석
  3. **시장 포지션** — 경쟁사 대비 현황 (표 1개)
  4. **4분기 전략 방향** — 3대 과제 (한 줄씩)
  5. **요청 사항** — 예산, 인력, 의사결정 (구체적 숫자)
  6. **리스크 & 대응** — 상위 3개만 (반드시 대응방안 함께)

**톤**: 자신감 있되 겸손하게. 성과는 명확히, 과제는 솔직히.
조현수 부사장은 "So What?"을 물어보는 분이므로, 모든 데이터에 해석과 액션을 붙일 것.
나쁜 소식을 숨기지 말되, 반드시 대응방안과 함께 제시.

**주의**: 부사장은 장황한 설명을 싫어합니다. 1페이지를 넘기면 읽지 않습니다.

---

### 보고서 3: 부하지시용 — 팀별 액션 아이템

**목적**: 각 팀장/팀원에게 4분기 액션 아이템을 지시

**형식 규격**:
- 분량: 팀별 반 페이지씩 (총 3~4페이지)
- 문체: 지시체 (~해 주세요, ~바랍니다) — 단, 각 수신자의 커뮤니케이션 스타일 반영
- 구조 (수신자별 반복):

  **[수신자 이름/직위/팀]에게**

  1. **배경 한줄** — 왜 이 지시를 하는지
  2. **핵심 지시사항** (번호 매기기)
     - 각 항목: 무엇을 / 언제까지 / 산출물 형태 / 협업 대상
  3. **기한** — 구체적 날짜
  4. **보고 방식** — 어떤 형태로 보고해야 하는지
  5. **참고사항** — 알아두면 좋은 맥락 정보

## 강사 노트

### 이 실습의 핵심 메시지

> **"같은 사실을 세 번 다르게 읽는 것이 임원의 역할입니다."**

하나의 전략보고서에 담긴 동일한 사실(매출, 점유율, 리스크)이 수신자에 따라 완전히 다른 형태로 변환됩니다. 이것이 바로 **임원 커뮤니케이션의 본질**입니다.

### 참가자에게 전달할 포인트 3가지

#### 1. 정보의 변환, 단순 요약이 아니다

| 나용 | 상사보고 | 부하지시 |
|------|---------|---------|
| "유럽 점유율 삼성에 역전당함" | "유럽 시장 포지셔닝 강화 필요" | "유럽 디오스 AI 캠페인 소재 10/4주 확정 바랍니다" |
| 사실 + 감정(위기감) | 사실 + 프레이밍(기회) | 사실 + 액션(구체적 행동) |

같은 사실이지만:
- **나에게는** 위기 신호로 읽힌다
- **상사에게는** 투자 근거로 변환된다
- **부하에게는** 구체적 행동 지침이 된다

#### 2. AI는 "변환 엔진"이지, "판단 주체"가 아니다

이 프롬프트가 잘 작동하는 이유는 **임원이 이미 판단 기준을 알고 있기 때문**입니다.

- 어떤 숫자가 중요한지 (KPI 선별)
- 누구에게 무엇을 강조해야 하는지 (수신자 분석)
- 어떤 톤이 적절한지 (조직 문화 이해)

AI가 하는 것은 이 판단을 **빠르게 문서화**하는 것입니다.
판단 자체는 여전히 임원의 몫입니다.

#### 3. 프롬프트 품질 = 조직 이해도

이 실습에서 조직정보 파일이 상세할수록 결과물 품질이 높아지는 것을 참가자가 직접 체감하게 됩니다.

이는 다음과 같은 시사점을 줍니다:
- **내 조직을 얼마나 잘 알고 있는가?** 가 AI 활용 품질을 결정한다
- 상사의 선호 스타일, 부하의 커뮤니케이션 방식을 명확히 정리할수록 AI 보조의 정확도가 높아진다
- 결국 **"좋은 프롬프트를 쓰는 임원 = 조직을 잘 아는 임원"** 이다

### 시연 순서 (권장)

1. **[30초]** 전략보고서(05번 파일)를 화면에 띄우고 분량 확인 — "이걸 읽고 3종 보고서를 만들어야 합니다"
2. **[30초]** 조직정보(04번 파일)를 간단히 보여줌 — "AI에게 조직 맥락을 알려줍니다"
3. **[1분]** 프롬프트를 복사·붙여넣기하고 파일 첨부 후 전송
4. **[2-3분]** AI가 3종 보고서를 생성하는 동안 설명:
   - "지금 AI가 같은 데이터를 세 가지 관점으로 변환하고 있습니다"
   - "여러분이 직접 하면 최소 2-3시간 걸릴 작업입니다"
5. **[3분]** 결과물 비교 리뷰:
   - 나용 보고서의 솔직한 톤 vs 상사보고의 격식체
   - 상사보고의 1페이지 제약 vs 부하지시의 상세함
   - 각 부하직원별 커뮤니케이션 스타일 반영 확인

### 예상 질문 & 답변

**Q: "조직정보를 매번 이렇게 정리해야 하나요?"**
A: 한 번 만들어두면 계속 재사용할 수 있습니다. 인사이동이 있을 때만 업데이트하면 됩니다. 오히려 이 작업 자체가 조직을 체계적으로 이해하는 과정입니다.

**Q: "실제 회사 정보를 AI에 넣어도 되나요?"**
A: 사내 AI 시스템(예: 사내 ChatGPT)이 있다면 가능합니다. 외부 AI 서비스 사용 시에는 회사 보안 정책을 반드시 확인하세요. 이 실습에서는 가상 데이터를 사용하는 이유이기도 합니다.

**Q: "AI가 만든 보고서를 그대로 쓰면 되나요?"**
A: 초안으로 활용하고 반드시 검토해야 합니다. 특히 숫자의 정확성, 조직 내 민감한 맥락, 정치적 뉘앙스는 임원이 직접 확인해야 합니다. AI는 80%를 빠르게 만들어주고, 나머지 20%를 임원이 다듬는 구조입니다.

## 1. 스킬이란?

### 개념 설명

```
[강의 멘트]

"여러분 회사에 SOP(표준 운영 절차)가 있죠?
월간 보고서 만들 때 항상 같은 순서로 하잖아요.

1. 데이터 수집
2. 분석
3. 차트 만들기
4. 보고서 포맷에 맞추기
5. 검토 후 제출

이 SOP를 AI에게 기억시키는 것이 '스킬'입니다.
한 번 만들어두면, 다음부터는 데이터만 바꿔 넣으면 됩니다.

SOP 문서는 서랍에 있고, 스킬은 AI 안에 있습니다.
둘 다 같은 역할인데, 스킬은 실행까지 자동으로 해줍니다."
```

### SOP vs 스킬 비교

| 구분 | 기존 SOP | Antigravity 스킬 |
|------|----------|-----------------|
| 형태 | 문서 (워드, PPT) | AI에 저장된 워크플로우 |
| 실행 | 사람이 단계별 수동 수행 | AI가 자동 실행 |
| 수정 | 문서 업데이트 후 재배포 | 스킬 편집 → 즉시 반영 |
| 공유 | 파일 공유, 교육 필요 | 스킬 공유 → 누구나 즉시 사용 |
| 일관성 | 담당자마다 품질 편차 | 항상 동일한 품질 |
| 소요 시간 | 수 시간 ~ 수 일 | 수 분 |

## 2. 스킬 저장하기

### Step-by-Step

#### Step 1: 워크플로우 완료 확인

```
[화면 설명]
Antigravity에서 프롬프트 실행이 완료된 상태.

```
[화면 설명]

#### Step 3: 스킬 정보 입력

```
[화면 설명]
스킬 저장 다이얼로그가 나타납니다.

| 입력 파라미터 | "분석 대상 데이터 파일" | 매번 바뀌는 입력값 정의 |
| 카테고리 | "보고서" / "분석" 등 | 정리용 태그 |

#### Step 4: 저장 완료

```
[화면 설명]

## 핵심 메시지 정리

```
[실습 마무리 시 반드시 전달]

1. "스킬 = SOP의 디지털 버전"
   - 종이 SOP: 절차만 적혀있고, 사람이 실행
   - AI 스킬: 절차가 저장되어 있고, AI가 실행

2. "한 번 만들면 계속 쓴다"
   - 이번 달에 3시간 걸린 보고서
   - 스킬 만드는 데 30분
   - 다음 달부터? 3분
   - 1년이면 30시간 이상 절약

3. "공유하면 팀 전체가 쓴다"
   - 나만의 노하우가 담긴 SOP를 팀에 공유
   - 신입사원도 베테랑과 동일한 품질의 산출물
   - 조직 역량의 표준화

4. "SOP를 만들 줄 아는 사람이 스킬도 만든다"
   - 업무를 잘 아는 사람이 최고의 스킬을 만듦
   - AI가 대체하는 것이 아니라, 전문가의 노하우를 증폭시키는 것
```
