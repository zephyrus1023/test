# CH05-CH06 마무리 강의노트

## 이 파트의 목적
- CH05에서는 사내 Vibe Coding 환경이 실제로 어떻게 연결되는지 짧게 보여준다.
- CH06에서는 오늘 실습을 다시 하나의 흐름으로 묶어주고, 개인 경험을 조직 적용으로 연결한다.
- 이 구간은 새로운 개념을 많이 가르치기보다, 오늘 배운 것을 `의미 있게 닫아주는 것`이 중요하다.

## CH05 Hi-D Code 소개 및 시연

### 강의 목표
- AI Studio에서 본 흐름이 사내 환경에서는 어떻게 이어질 수 있는지 보여준다.
- 외부 도구 체험이 끝나고 나면, 참가자가 자연스럽게 `우리 조직에서는 어디서 이걸 이어갈 수 있지?`를 떠올리게 만든다.

### 핵심 포인트
- 발표자는 `최남석(Agentic AI 팀)`이다.
- 여기서는 기능을 세세하게 설명하기보다, `사내 Vibe Coding 환경이 이미 준비되고 있다`는 메시지가 중요하다.
- 이미지상 불량 검출 S/W 사례는 “이런 흐름이 실제 문제 해결로 이어진다”는 대표 장면으로 보여주면 된다.

### 강사 연결 멘트
- 앞에서는 공개 도구와 브라우저 환경에서 실험해봤다면, 이제는 이런 흐름이 사내 환경에서 어떻게 이어질 수 있는지 보겠습니다.
- 핵심은 도구 이름보다도, 우리 조직 안에서 실제 실행 가능한 환경으로 연결된다는 점입니다.

### 짧은 정리 멘트
- 결국 중요한 것은 AI를 한 번 잘 써보는 것이 아니라, 내 환경과 조직의 환경에서 반복 가능하게 만드는 것입니다.

## CH06-1 오늘의 핵심 정리

### 강의 목표
- 오늘 수업을 `도구 목록`이 아니라 `하나의 흐름`으로 다시 묶는다.
- 참가자가 “오늘 내가 무엇을 경험했는가”를 다시 말할 수 있게 만든다.

### 진행 포인트
- 오늘 실제로 경험한 5단계를 표 기준으로 다시 짚는다.
- 세부 기능보다도 연결을 강조한다.
	- AI 트렌드
	- Gemini / ChatGPT / GPTs
	- NotebookLM
	- AI Studio Build
	- Hi-D Code

### 강조할 문장
- 오늘 우리는 AI를 읽고, 맡기고, 맥락으로 깊게 만들고, 실제 앱으로 구현해봤습니다.
- 각 도구는 따로 노는 것이 아니라, 앞 단계의 결과가 다음 단계의 입력이 되는 구조로 연결됩니다.

### 마지막 질문
- 내 조직에서 AI를 “한 번 써보는 도구”로 둘 것인가, 아니면 맥락을 쌓고 반복 가능한 흐름으로 남기는 실행 체계로 만들 것인가?

## CH06-2 조직에서의 다음 단계

### 강의 목표
- 오늘 배운 것을 조직에 어떻게 옮길지 현실적인 다음 행동으로 바꾼다.
- 거창한 혁신보다도 `작게 시작해서 남기고 공유하는 것`이 중요하다는 메시지를 준다.

### 설명 흐름
- 개인
	- 자기 업무 하나를 AI로 다시 해본다.
- 팀
	- 잘 된 프롬프트, 노트북, 결과물을 공유한다.
- 조직
	- API key, 보안, 데이터 경계, 사내 실행 환경을 정리한다.

### 리더에게 던질 문장
- AI는 잘 쓰는 사람 몇 명으로 끝나지 않습니다. 공유 가능한 형식으로 남길 때 조직 역량이 됩니다.
- 문제 선정, 맥락 자산화, 운영 원칙 세 가지가 갖춰져야 확산이 됩니다.

### 마무리 멘트
- 오늘의 교육은 끝나지만, 실제 변화는 각자 돌아가서 하나를 다시 해보는 순간부터 시작됩니다.
- 작은 성공 사례 3개를 팀 안에 쌓으면, 그다음부터는 교육이 아니라 문화가 됩니다.
