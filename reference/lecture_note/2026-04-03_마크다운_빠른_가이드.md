# 수업용 마크다운 빠른 가이드

## 1. 왜 마크다운을 쓰나

- 마크다운은 사람이 읽기 쉽고, AI도 구조를 파악하기 쉬운 형식이다.
- 제목, 불릿, 코드블록처럼 경계가 분명해서 저장·복사·재사용에 유리하다.
- Gemini, ChatGPT, NotebookLM 모두 이런 구조를 좋아한다.

## 2. 수업에서 먼저 보여줄 6가지

### 제목

```markdown
# 큰 제목
## 중간 제목
### 작은 제목
```

### 불릿

```markdown
- 첫 번째 항목
- 두 번째 항목
- 세 번째 항목
```

### 번호 목록

```markdown
1. 첫 단계
2. 두 번째 단계
3. 세 번째 단계
```

### 강조

```markdown
**중요한 단어**
*가볍게 강조*
```

### 코드블록

````markdown
```markdown
# 제목
- 항목 1
- 항목 2
```
````

### 표

```markdown
| 항목 | 내용 |
|------|------|
| 역할 | 전략기획 |
| 목적 | 회의 요약 |
| 출력 | 브리프 |
```

## 4. 강사가 이렇게 말하면 좋다

- “마크다운은 코딩 문법이라기보다, 글을 구조화하는 가장 쉬운 방법입니다.”
- “제목과 불릿만 잘 써도 AI가 훨씬 안정적으로 이해합니다.”
- “특히 코드블록으로 감싸 두면, 나중에 다른 AI에 그대로 다시 넣기가 좋습니다.”

## 5. 바로 해보세요 멘트

- “지금 자기소개를 그냥 문장으로 쓰지 말고, 제목과 불릿으로 나눠 써보세요.”
- “방금 나온 회의 요약을 코드블록 안에 다시 넣어달라고 요청해보세요.”
- “지금 정리한 내용을 다음 도구에 그대로 넣는다고 생각하고 써보면 됩니다.”

## 6. 참가자에게 바로 던질 연습 과제

### 과제 1
- 자기소개를 `기본 정보 / 주요 경력 / 관심 분야 / 교육 목표` 네 섹션으로 나눠 써보기

### 과제 2
- 회의록 내용을 `핵심 이슈 / 결정사항 / 액션 아이템` 세 섹션으로 다시 정리해보기

### 과제 3
- 리서치 결과를 `핵심 메시지 / 근거 데이터 / 시사점` 형식으로 바꿔보기

## 7. 한 문장 정리

- 마크다운은 예쁘게 쓰기 위한 문법이 아니라, **AI가 좋아하는 방식으로 생각과 자료를 구조화하는 습관**이다.
