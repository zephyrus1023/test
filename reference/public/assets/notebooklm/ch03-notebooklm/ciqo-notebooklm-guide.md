# CIQO와 NotebookLM 교차 분석 가이드

## CIQO란

- `C = Context`
  - 내부 회의 맥락, KPI, 현재 문제의식처럼 분석의 출발점이 되는 자료
- `I = Issue`
  - 지금 무엇이 핵심 이슈인지, 무엇을 판단해야 하는지 정리한 문제 프레이밍
- `Q = Question`
  - 어떤 관점으로 물어볼지 정한 질문 구조
- `O = Output`
  - 누구를 위한 결과물인지에 맞춘 출력 형식

## 왜 중요한가

- 같은 보고서를 넣어도, 어떤 맥락과 이슈를 먼저 붙이느냐에 따라 해석이 달라진다.
- 내부 회의 맥락 없이 WEF와 Deloitte만 보면 일반론에 머무르기 쉽다.
- 반대로 내부 회의 자료만 보면 외부 시장 변화와 경쟁 기준이 부족해질 수 있다.
- NotebookLM은 이 둘을 함께 묶어 `출처 기반`으로 비교하게 해준다는 점이 강점이다.

## 이번 실습에서의 적용 방식

### Context
- AX 전략 회의 맥락 문서
- 회의록 요약
- ChatGPT Deep Research 메모

### Issue
- EXAONE 도입률 목표 미달
- AI/ML 인력 충원 병목
- 외부 LLM 의존도 상승

### Question
- WEF 보고서가 위 이슈 중 무엇을 설명해 주는가
- Deloitte 보고서는 시장/산업 전환 측면에서 어떤 추가 맥락을 주는가
- 내부 회의에서 놓친 시사점은 무엇인가

### Output
- 임원 브리핑 1페이지
- 후속 액션 아이템
- NotebookLM Studio 슬라이드 초안

## 기억할 점

- 좋은 NotebookLM 분석은 `자료 개수`보다 `자료 조합`이 좌우한다.
- 내부 자료와 외부 자료를 함께 넣어야 다층적 해석이 가능하다.
- 결국 NotebookLM의 가치는 “많이 읽는 것”이 아니라 “출처를 가진 비교 분석”에 있다.
