# CH03-1 Gemini 일반 채팅 시연 - 단일 흐름 프롬프트 세트

> **세션**: CH03-1 Gemini 일반 채팅 (단일 흐름 시연)
> **소요 시간**: 약 20~25분
> **도구**: Gemini Advanced (gemini.google.com)
> **흐름**: 음성파일 업로드 → 회의록 생성 → 자료첨부 멀티턴 → 전략추론 → 딥리서치
> **사전 준비물**:
> - `01_가상회의_대본_TTS.md` 기반 생성된 음성 파일 (MP3/WAV)
> - `02_회의_맥락_참고자료.md` (브리핑 데이터 문서)

---

## 시연 개요

하나의 Gemini 대화 세션 안에서 5단계를 연속으로 진행합니다.
임원이 실제 업무에서 겪는 시나리오 -- "회의 녹음 → 정리 → 분석 → 의사결정 → 심층 조사"를 AI 하나로 끝내는 경험을 보여줍니다.

### 프롬프트 엔지니어링 4원칙 매핑

| 원칙 | 설명 | 이 시연에서의 적용 |
|------|------|-------------------|
| **역할(Role)** | AI에게 전문가 페르소나 부여 | "C-Suite 전략 보좌관", "경영 전략 애널리스트" |
| **맥락(Context)** | 배경 정보와 상황 제공 | 회의 음성, 참고자료 파일 첨부로 맥락 주입 |
| **형식(Format)** | 출력 형태 지정 | 표, 불렛포인트, 프레임워크 등 구조화된 형식 요청 |
| **단계(Step)** | 작업 순서를 명시 | 5단계 연속 흐름 자체가 단계적 사고의 예시 |

## Step 1: 음성 → 회의록 생성

> **소요 시간**: 약 4분 (업로드 1분 + 생성 2분 + 설명 1분)
> **4원칙 포인트**: **역할** (전문 비서) + **형식** (구조화된 회의록 템플릿)

### 조작 순서
1. Gemini Advanced 새 대화 시작
2. 클립 아이콘 클릭 → 음성 파일 업로드
3. 아래 프롬프트 입력 후 전송

### 프롬프트

```
첨부한 음성 파일은 LG그룹 하반기 AX(AI 전환) 전략 회의 녹음입니다.

이 음성을 듣고 아래 형식으로 구조화된 회의록을 작성해 주세요.

## 출력 형식

### 1. 회의 기본 정보
- 회의명, 일시, 참석자(직책 포함)

### 2. 안건별 논의 내용
각 안건마다:
- **안건 제목**
- **발언 요약**: 누가 무슨 말을 했는지 핵심만
- **쟁점**: 의견이 갈린 부분

### 3. 결정사항
- 번호 매기기, 담당자 명시

### 4. 후속조치(Action Items)
| 번호 | 조치 내용 | 담당자 | 기한 |
|------|----------|--------|------|

### 5. 미결 사항
- 추가 논의가 필요한 이슈
```

### 강사 노트

```
[타이밍] 음성 업로드 후 Gemini가 처리하는 동안 30초~1분 대기 발생.
이 시간을 활용하여 설명:

- "지금 Gemini가 음성을 텍스트로 변환하고, 동시에 내용을 이해해서
  구조화하고 있습니다. 멀티모달의 핵심이 바로 이것입니다."

3. 회의록 품질이 실무 수준임을 확인

[4원칙 연결]
```

## Step 2: 회의록 기반 핵심 이슈 추출

> **소요 시간**: 약 3분 (프롬프트 1분 + 생성 1분 + 설명 1분)
> **4원칙 포인트**: **역할** (전략 애널리스트) + **단계** (분석 → 평가 → 우선순위)

### 조작 순서
1. 같은 대화에서 이어서 프롬프트 입력 (멀티턴)

### 프롬프트

```
방금 작성한 회의록을 기반으로, 경영 전략 애널리스트 관점에서 분석해 주세요.

## 요청 사항

### 1. 핵심 전략 이슈 TOP 3
각 이슈마다:
- **이슈명**: 한 줄 요약
- **배경**: 왜 이것이 중요한지
- **회의에서의 논의 수준**: 충분 / 부분적 / 미흡
- **리스크 수준**: 🔴 높음 / 🟡 중간 / 🟢 낮음

### 2. 의사결정 갭 분석
- 결정은 됐지만 실행 방안이 불명확한 항목
- 논의는 됐지만 결정이 안 된 항목
- 아예 다뤄지지 않았지만 다뤄졌어야 할 항목

### 3. CEO 어젠다 추천
- 다음 회의에서 반드시 다뤄야 할 안건 2~3개
- 각 안건의 예상 소요 시간과 필요 참석자
```

### 강사 노트

```
[타이밍] Step 1의 결과물이 화면에 있는 상태에서 바로 이어서 진행.

[핵심 교수 포인트]
1. "같은 대화창에서 이어가는 것"의 의미 강조
   → "AI가 앞의 회의록을 기억하고 있습니다. 이것이 '멀티턴'입니다."
   → "새 대화를 열면 맥락이 사라집니다. 하나의 흐름을 유지하세요."

2. 역할 전환에 주목시킬 것
   → "Step 1에서는 '비서'였고, Step 2에서는 '전략 애널리스트'로 바뀌었습니다."
   → "같은 AI에게 다른 관점으로 보라고 요청하는 것입니다."

3. '의사결정 갭 분석' 결과가 나오면 반응 유도
   → "여기 '다뤄지지 않았지만 다뤄졌어야 할 항목'을 보세요.
      AI가 회의의 빈 곳을 찾아냈습니다."

[4원칙 연결]
- 역할: "경영 전략 애널리스트 관점"으로 명시적 역할 부여
- 단계: 이슈추출 → 갭분석 → 어젠다추천의 3단계 구조
- 형식: 리스크 수준에 색상 이모지, 표 형식 활용
```

## Step 3: 참고자료 첨부 + 멀티턴 분석

> **소요 시간**: 약 5분 (업로드 1분 + 생성 2분 + 설명 2분)
> **4원칙 포인트**: **맥락** (외부 자료로 맥락 확장) + **단계** (교차 분석)

### 조작 순서
1. 같은 대화에서 클립 아이콘 클릭 → `02_회의_맥락_참고자료.md` 업로드
2. 아래 프롬프트 입력 후 전송

### 프롬프트

```
방금 첨부한 문서는 이번 회의의 배경이 되는 브리핑 자료입니다.
(경쟁사 동향, 시장 데이터, 내부 KPI 등)

이 참고자료와 앞서 정리한 회의록을 교차 분석해서 다음을 정리해 주세요.

## 요청 사항

### 1. 회의 논의 vs 실제 데이터 대조표
| 회의에서 언급된 내용 | 참고자료의 실제 데이터 | 일치 여부 | 비고 |
|---------------------|----------------------|----------|------|

### 2. 계획 대비 실적 갭
- 상반기 목표 대비 실제 달성률
- 경쟁사 대비 상대적 위치
- 가장 큰 괴리가 발생한 영역과 원인 분석

### 3. 회의에서 놓친 포인트
- 참고자료에는 있지만 회의에서 전혀 논의되지 않은 중요 데이터
- 해당 데이터가 의사결정에 미칠 수 있는 영향

### 4. 데이터 기반 수정 권고
- 회의 결정사항 중 참고자료 데이터에 비춰 재검토가 필요한 항목
- 수정 방향 제안
```

### 강사 노트

```
[타이밍] 파일 업로드 시 잠깐 대기. 이때 설명:

- "지금 AI에게 '새로운 눈'을 달아주는 겁니다.
  회의 내용만 있었을 때와 실제 데이터를 함께 볼 때,
  분석의 깊이가 완전히 달라집니다."

[핵심 교수 포인트 - 이 Step이 가장 중요!]
1. 멀티턴 + 파일 첨부의 위력을 보여주는 핵심 장면
   → "한 대화 안에서 음성 파일, 텍스트 분석, 문서 첨부까지.
      세 가지 입력을 하나의 맥락으로 통합 처리합니다."

2. '회의에서 놓친 포인트' 결과에서 임원들 반응 끌어내기
   → "이 부분이 실무에서 가장 가치 있습니다.
      100페이지 보고서를 회의 전에 다 읽기 어렵잖아요.
      AI가 '여기 빠뜨리셨습니다'라고 알려줍니다."

3. 프롬프트 엔지니어링 4원칙 중 '맥락(Context)' 강조
   → "프롬프트가 아무리 좋아도, 맥락이 빈약하면 결과도 빈약합니다.
      파일 첨부 = 맥락 주입입니다."

[4원칙 연결]
- 맥락: 참고자료 파일 첨부로 AI의 분석 맥락을 대폭 확장
- 형식: 대조표(표) + 갭 분석(불렛) + 권고(구조화)
- 단계: 대조 → 갭 분석 → 누락 발견 → 수정 권고의 4단계
```
