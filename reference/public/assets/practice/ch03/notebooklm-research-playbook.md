# CH04 NotebookLM 시연 - 프롬프트 및 활용 가이드

> **세션**: CH04 NotebookLM (Fast Research + Note→Source 전환)
> **소요 시간**: 약 20~25분
> **도구**: NotebookLM (notebooklm.google.com)
> **흐름**: 소스 업로드 → 다중 소스 질의 → 패스트 리서치 → 노트→소스 전환 → 스튜디오 패널 활용
> **사전 준비물**:
> - WEF 미래 직업 보고서 2025 (PDF)
> - 딜로이트 글로벌 럭셔리 보고서 2026 (PDF)
> - (선택) 추가 보고서 1~2건

---

## 1. 소스 업로드 안내

> **소요 시간**: 약 3분
> **목적**: NotebookLM 노트북을 생성하고 분석 대상 소스를 업로드

### 조작 순서

1. NotebookLM (notebooklm.google.com) 접속
2. **"새 노트북"** 클릭
3. 노트북 이름 설정: `AX Camp 전략 리서치`
4. **"소스 추가"** → 파일 업로드 선택
5. 아래 파일들을 순서대로 업로드

### 업로드할 소스 목록

| 순번 | 파일명 | 출처 | 페이지 수 | 용도 |
|------|--------|------|----------|------|
| 1 | `WEF_Future_of_Jobs_Report_2025.pdf` | 세계경제포럼(WEF) | 135쪽 | AI 시대 일자리/스킬 변화 데이터 |
| 2 | `global-powers-of-luxury-goods-2026.pdf` | 딜로이트 | 75쪽 | 글로벌 럭셔리 산업 트렌드 |
| 3 | (선택) `Report on the Space Economy 2025.pdf` | 유럽우주국(ESA) | 358쪽 | 우주경제 시장 데이터 |

### 강사 노트

```
[타이밍] 업로드 중 1~2분 소요. 이 시간에 설명:

- "NotebookLM은 업로드한 문서만을 소스로 사용합니다.
  인터넷 검색이 아니라 여러분이 준 자료 안에서만 답합니다.
  그래서 '할루시네이션(거짓 정보)'이 거의 없습니다."

- "각 답변에 출처 번호가 달립니다. 클릭하면
  원문 해당 부분으로 바로 이동합니다."

[핵심 교수 포인트]
1. NotebookLM vs 일반 챗봇의 차이
   → "ChatGPT/Gemini는 '세상 모든 것을 아는 척'합니다.
      NotebookLM은 '내가 준 자료만 봅니다'. 이게 비즈니스에서 더 안전합니다."

2. 소스 개수 제한 안내
   → "노트북 하나에 소스 50개까지 업로드 가능합니다.
      PDF, 웹페이지, YouTube 영상, Google Docs 등 다양한 형식을 지원합니다."
```

## 2. 기본 질문 프롬프트 (다중 소스 비교 분석)

> **소요 시간**: 약 5분 (프롬프트 3개 순차 진행)
> **목적**: 여러 소스를 교차 분석하는 NotebookLM의 핵심 가치 시연

### 프롬프트 2-1: 단일 소스 심층 질의

```
WEF 미래 직업 보고서에서 AI로 인해 가장 크게 성장할 것으로 전망되는
직무 TOP 5와, 가장 크게 감소할 직무 TOP 5를 정리해 줘.

각 직무별로:
- 2025-2030 예상 변화율 (%)
- 변화의 주요 원인
- 해당 직무가 많은 산업군
을 표로 만들어 줘.
```

### 프롬프트 2-2: 다중 소스 교차 분석

```
WEF 보고서의 '기술 변화가 산업에 미치는 영향' 분석과
딜로이트 럭셔리 보고서의 '디지털 전환 트렌드'를 교차 비교해 줘.

두 보고서에서 공통으로 강조하는 트렌드는 무엇이고,
서로 다른 시각을 보이는 부분은 무엇인지 정리해 줘.
```

### 프롬프트 2-3: 비즈니스 임팩트 분석

```
두 보고서의 내용을 종합했을 때,
LG그룹 같은 대기업 임원이 주목해야 할 시사점 3가지를 뽑아줘.

각 시사점마다:
- 근거 데이터 (어느 보고서 몇 페이지)
- 임원 액션 아이템
- 리스크와 기회
를 구분해서 정리해 줘.
```

### 강사 노트

```
[타이밍]
- 프롬프트 2-1: 2분 (입력 + 결과 확인)
- 프롬프트 2-2: 2분 (교차 분석 결과 + 출처 클릭 시연)
- 프롬프트 2-3: 1분 (결과 확인 + 설명)

[핵심 교수 포인트]
1. 프롬프트 2-1 결과 나오면 → 출처 번호 클릭 시연
   → "이 [1] 표시를 클릭하면 원문 해당 페이지로 갑니다.
      AI가 만들어낸 게 아니라 실제 데이터인지 바로 검증 가능합니다."

2. 프롬프트 2-2에서 다중 소스의 가치 강조
   → "135페이지짜리 보고서와 75페이지짜리 보고서를
      교차 분석하라고 했습니다. 사람이 하면 며칠 걸릴 작업입니다."

3. 프롬프트 2-3에서 '임원 관점' 프레이밍
   → "같은 데이터도 '임원이 주목해야 할'이라고 프레이밍하면
      전혀 다른 관점의 답변이 나옵니다. 이것이 역할 설정의 힘입니다."
```

## 3. 패스트 리서치 프롬프트

> **소요 시간**: 약 4분
> **목적**: NotebookLM의 Fast Research 기능으로 웹 소스 자동 수집 시연

### 조작 순서

1. NotebookLM 좌측 소스 패널에서 **"소스 추가"** 클릭
2. **"패스트 리서치(Fast Research)"** 또는 **"웹 검색"** 옵션 선택
3. 아래 리서치 주제 입력
4. NotebookLM이 자동으로 관련 웹 소스를 수집하여 노트북에 추가

### 패스트 리서치 주제

```
2025-2026년 글로벌 AI 가전 시장 트렌드와 주요 기업(LG, 삼성, 하이센스)의
AI 전략 비교 분석
```

### 대안 주제 (시간/상황에 따라 선택)

```
한국 대기업의 AI 인재 양성 전략과 리스킬링 프로그램 최신 동향
```

### 강사 노트

```
[타이밍] Fast Research가 소스를 수집하는 데 1~2분 소요.

[조작 포인트]
- Fast Research가 여러 웹 소스를 자동으로 찾아 추가하는 과정을 보여줄 것
- 수집된 소스 목록이 나오면 어떤 출처들이 포함됐는지 확인

[핵심 교수 포인트]
1. Fast Research의 개념
   → "기존에는 소스를 하나씩 직접 업로드했습니다.
      Fast Research는 AI가 주제에 맞는 웹 자료를 자동으로 찾아서
      노트북에 추가해 줍니다. 리서치 자동화입니다."

2. 기존 소스와의 통합
   → "이미 업로드한 WEF/딜로이트 보고서 + 방금 수집된 웹 소스가
      하나의 노트북에 공존합니다. 이제 이 전체를 교차 분석할 수 있습니다."

3. 품질 검증의 중요성
   → "AI가 찾아온 소스가 신뢰할 만한지는 여러분이 판단해야 합니다.
      출처를 반드시 확인하세요."
```

## 4. 노트 → 소스 전환 가이드

> **소요 시간**: 약 4분
> **목적**: 대화 내용을 노트로 저장하고, 그 노트를 다시 소스로 전환하는 고급 기능 시연

### Step 4-1: 대화를 노트로 저장

**조작 순서:**
1. 앞서 진행한 질의응답 중 가장 유용했던 답변을 선택
2. 해당 답변 우측의 **"노트에 저장"** (핀 아이콘 또는 저장 버튼) 클릭
3. 노트 패널에서 내용 확인

**설명 프롬프트 (강사가 구두로 설명):**

```
"지금 AI가 정리해 준 '임원 시사점 3가지'가 유용하니까,
이걸 노트로 저장하겠습니다.
→ [답변 옆 저장 버튼 클릭]
→ 노트 패널에 저장된 것 확인"
```

### Step 4-2: 노트를 소스로 전환

**조작 순서:**
1. 좌측 노트 패널에서 저장된 노트 선택
2. 노트 상단 또는 우클릭 메뉴에서 **"소스로 전환"** 클릭
3. 해당 노트가 소스 목록에 추가된 것 확인

**전환 후 활용 프롬프트:**

```
방금 소스로 전환된 '임원 시사점 분석' 노트와
원래 WEF/딜로이트 보고서를 함께 참조해서,

각 시사점별 실행 계획을 구체화해 줘.
특히 LG그룹 임원이 다음 분기에 바로 착수할 수 있는
액션 아이템 중심으로 정리해 줘.
```

### 강사 노트

```
[타이밍]
- Step 4-1 (노트 저장): 1분
- Step 4-2 (소스 전환): 1분
- 전환 후 활용 프롬프트: 2분

[핵심 교수 포인트 - 이 기능이 NotebookLM의 킬러 피처]
1. 지식의 누적 구조
   → "AI와의 대화 결과를 노트로 저장 → 소스로 전환하면,
      이전 분석 결과가 새로운 분석의 기반이 됩니다.
      지식이 쌓이는 겁니다."

2. 실무 적용 시나리오
   → "매주 회의 후 분석 결과를 노트로 저장 → 소스로 전환
      → 다음 회의 때 이전 분석을 참조하며 새로운 분석 수행
      → 분기별 전략 리뷰에서 누적된 인사이트 종합"

3. 일반 챗봇과의 결정적 차이
   → "ChatGPT나 Gemini에서는 대화가 끝나면 지식이 사라집니다.
      NotebookLM은 노트 → 소스 전환으로 지식을 영구 보관할 수 있습니다.
      이게 '개인 지식 베이스'입니다."
```

## 5. 스튜디오 패널 활용 프롬프트

> **소요 시간**: 약 4분
> **목적**: NotebookLM 스튜디오에서 슬라이드/팟캐스트 등 콘텐츠 자동 생성 시연

### 조작 순서

1. 우측 **"스튜디오"** 패널 열기
2. 생성할 콘텐츠 유형 선택

### 5-A: 슬라이드 생성 프롬프트

**스튜디오 패널에서 "슬라이드" 선택 후 커스터마이징 입력:**

```
AI 시대 임원 리더십 전략 - 핵심 데이터 요약

대상 청중: LG그룹 C-Level 임원
슬라이드 수: 8~10장
스타일: 간결하고 데이터 중심, 경영진 보고 형식
포함 내용:
- WEF 보고서의 핵심 수치
- 딜로이트 보고서의 시장 트렌드
- 임원 액션 아이템 요약
```

### 5-B: 팟캐스트(오디오 오버뷰) 생성 프롬프트

**스튜디오 패널에서 "오디오 오버뷰" 선택 후 커스터마이징 입력:**

```
두 명의 전문가가 AI 시대의 기업 전략에 대해 대화하는 형식으로 만들어 줘.
특히 LG그룹 같은 한국 대기업의 관점에서 논의해 줘.
핵심 데이터를 자연스럽게 대화에 녹여서 설명해 줘.
```

### 강사 노트

```
[타이밍]
- 슬라이드 또는 팟캐스트 중 하나만 시연 (시간에 따라 선택)
- 슬라이드: 생성 1~2분 + 결과 확인 1분
- 팟캐스트: 생성 시작만 보여주고 설명 (완성까지 수 분 소요)

[핵심 교수 포인트]
1. "입력은 보고서, 출력은 프레젠테이션"
   → "200페이지가 넘는 보고서 2편이 10장짜리 슬라이드로 변환됩니다.
      발표 준비 시간이 획기적으로 줄어듭니다."

2. 팟캐스트 시연 시
   → "이 오디오를 출퇴근길에 들으면,
      200페이지 보고서의 핵심을 15분 만에 파악할 수 있습니다."

3. 비즈니스 활용 예시
   → "이사회 브리핑 자료, 팀 공유용 요약, 교육 콘텐츠 등
      원본 소스 하나로 다양한 포맷의 산출물을 만들 수 있습니다."
```

## 6. 초급/상급 분기 가이드

> **목적**: 수강생 수준에 따라 시연 깊이를 조절하기 위한 강사용 분기 안내

### 초급 코스 (A Course) - 기본 옵션 클릭 중심

| 단계 | 활동 | 조작 | 소요 시간 |
|------|------|------|----------|
| 1 | 소스 업로드 | 파일 드래그앤드롭 | 2분 |
| 2 | 추천 질문 활용 | NotebookLM이 자동 생성한 추천 질문 클릭 | 2분 |
| 3 | 오디오 오버뷰 | 스튜디오 → 오디오 오버뷰 → 기본 설정으로 생성 | 2분 |
| 4 | 노트 저장 | 유용한 답변 → "노트에 저장" 클릭 | 1분 |
| **합계** | | | **~7분** |

**강사 멘트 (초급):**

```
"NotebookLM은 복잡한 프롬프트 없이도 사용할 수 있습니다.
보고서를 올리면 AI가 알아서 추천 질문을 만들어 줍니다.
그 질문을 클릭하기만 하면 됩니다.
오디오 오버뷰도 버튼 하나면 자동 생성됩니다."
```

| 단계 | 활동 | 조작 | 소요 시간 |
|------|------|------|----------|
| 1 | 소스 업로드 + 패스트 리서치 | 파일 업로드 + Fast Research로 웹 소스 자동 수집 | 3분 |
| 2 | 다중 소스 교차 분석 | 직접 작성한 프롬프트로 교차 분석 질의 | 4분 |

**커스터마이징 포인트 상세:**

#### 슬라이드 커스터마이징

```
[기본] 스튜디오 → 슬라이드 → "생성" 클릭 (초급)

[커스터마이징] 아래 내용을 입력 (상급):
- 청중: "LG그룹 사업부장급 임원 (기술 배경 없음)"
- 스타일: "맥킨지 스타일, 데이터 시각화 중심"
- 분량: "핵심 메시지 5장 + 부록 데이터 3장"
- 특별 요청: "각 슬라이드에 'So What' 한 줄 포함"
```

#### 팟캐스트 커스터마이징

```

[커스터마이징] 아래 내용을 입력 (상급):
- 대화 형식: "AI 전략 전문가와 현업 임원의 대담"

- 길이: "10분 내외"
- 특별 요청: "각 보고서의 핵심 수치를 대화에 자연스럽게 포함"
```

### 강사 노트 (분기 안내)

```
[분기 판단 기준]
- 초급: 사전 진단 결과 A Course / AI 도구 사용 경험 거의 없음
  → 추천 질문 클릭, 기본 생성 버튼 위주로 진행
  → "어렵지 않죠? 버튼만 누르면 됩니다"

- 상급: 사전 진단 결과 B Course / AI 도구 사용 경험 있음
  → 직접 프롬프트 작성, 커스터마이징 옵션 활용
  → "이제 여러분만의 방식으로 AI를 조종할 수 있습니다"

[시간 조절]
- 시간 여유 있음 → 초급도 노트→소스 전환까지 진행
- 시간 부족 → 상급도 슬라이드 OR 팟캐스트 중 택 1

[핵심 메시지]
- 초급: "AI는 어렵지 않습니다. 클릭 몇 번이면 됩니다."
- 상급: "AI의 진짜 가치는 커스터마이징에서 나옵니다.
        여러분의 맥락을 넣을수록 결과가 좋아집니다."
```

## 비상 대응 (시연 중 문제 발생 시)

| 상황 | 대응 |
|------|------|
| PDF 업로드 실패 | 파일 크기 확인 (200MB 제한). Google Drive에서 직접 연결 시도 |
| 패스트 리서치 작동 안 함 | "이 기능은 최신 업데이트에서 제공됩니다" 설명 후, 수동으로 웹 URL 소스 추가 |
| 오디오 오버뷰 생성 안 됨 | "영어 소스일 때 가장 잘 작동합니다" 설명. 미리 만들어 둔 샘플 오디오 재생 |
| 응답 속도 느림 | 소스 개수를 줄여서 재시도. 특정 소스만 선택하여 질의 |
| 노트→소스 전환 메뉴 없음 | NotebookLM 버전 차이일 수 있음. 노트 내용을 복사 → 텍스트 소스로 직접 추가 |
| 출처 번호 클릭 안 됨 | 브라우저 새로고침 후 재시도. 원문 표시 기능을 수동으로 검색 |
